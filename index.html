We all know that JavaScript is an interpreter programming language. Behind the scene all JavaScript source code execute using JavaScript Engine. JavaScript Engine is an another program which can read, understand and translate JavaScript source code to machine code. First of all JavaScript source code will go to the Parser (parser is a program that read code link by link). Then Parser jobs is to make Abstract Window Tree then this tree will convert to machine code and then machine code will start execution. The browser's JavaScript engine creates a special environment to handle the transformation and execution of JavaScript code. This environment is known as the Execution Context. There are two kind of execution context in JavaScript. 1)Global Execution Context and 2)Function Execution Context. Each execution context has two phase. 1)Creation Phase and 2)Execution Phase. Creation Phase job is to collect all the declaration of variable and function. And Execution Phase job is assign value to all declared variable and call the function.

Local storage and Session storage are often the same, with only a few basic differences which are, the data in the local storage is stored for a long time (i.e., must be deleted via JavaScript or manually) but the data is the session storage is stored until the browser or current tab is closed. As the data in the local storage is storage for a long time so we can get all data from local storage and display on UI. We can also get data from session storage and display on UI but we have to set and get data until browser or current tab is closed. Session storage data will be lost after closing the browser or current tab. The storage capacity of local storage is 5MB or 10MB and the storage capacity of session storage is 5MB. Local storage data shared between all tabs and windows from the same origin. Session storage data survives page refresh.
